## 使用方法

### 启动系统

1. **启动数据库**（Docker版）
   ```bash
   # 双击运行
   start_all_docker.bat
   ```

2. **访问前端**
   - 打开浏览器访问：`http://localhost:8000/`
   - 或直接打开：`hospital-edc-frontend/index.html`

3. **登录系统**
   - 用户名：`admin`
   - 密码：`Admin@123`

### 数据录入流程

1. **选择患者**
   - 点击侧边栏"患者管理"
   - 选择要录入数据的患者

2. **创建访视**
   - 点击"新建访视"按钮
   - 选择访视类型（基线/M6/M12/M18/M24）
   - 选择访视日期

3. **录入数据**
   - 点击侧边栏"数据录入"
   - 切换不同Tab录入各项数据：
     - 体格检查
     - 实验室检查
     - 合并症
     - 用药记录
     - 费用指标
     - PHQ-9问卷
     - GAD-7问卷
     - 饮食评估
     - 运动评估
     - 膳食调查

4. **保存数据**
   - 点击"保存草稿"：保存为草稿状态
   - 点击"提交审核"：提交给质控员审核

## 功能特点

### 智能计算
- **BMI自动计算**：输入身高体重后自动计算
- **腰臀比自动计算**：输入腰围臀围后自动计算
- **费用自动汇总**：各项费用自动求和

### 量表评分
- **PHQ-9抑郁量表**：自动计算总分并分级（无/轻度/中度/中重度/重度）
- **GAD-7焦虑量表**：自动计算总分并分级（无/轻度/中度/重度）
- **饮食评估**：自动计算总分并分级（差/尚可/一般/良好）
- **运动评估**：自动计算总分并分级（差/尚可/一般/良好）

### 用户体验
- **Tab切换**：清晰的模块划分，便于数据录入
- **实时反馈**：输入数据后立即显示计算结果
- **视觉提示**：不同分级使用不同颜色标识
- **响应式设计**：适配不同屏幕尺寸

## 技术栈

- **前端框架**: 原生JavaScript（无框架）
- **UI库**: Tailwind CSS（CDN）
- **HTTP客户端**: Fetch API
- **数据存储**: LocalStorage（token）

## 注意事项

1. **数据安全**
   - 所有API请求都需要携带token
   - Token存储在LocalStorage中
   - 登出时清除token

2. **数据验证**
   - 前端进行基础验证
   - 后端进行完整验证
   - 不要完全信任前端数据

3. **性能优化**
   - 页面采用懒加载
   - 只在需要时加载HTML内容
   - 避免重复初始化

4. **浏览器兼容性**
   - 推荐使用Chrome/Edge最新版
   - 需要支持ES6+
   - 需要支持Fetch API

## 常见问题

### Q: 页面加载失败
**A**: 检查是否正确启动了后端服务，确认API地址配置正确

### Q: 数据保存失败
**A**: 检查网络连接，查看浏览器控制台错误信息

### Q: 自动计算不工作
**A**: 确保输入了有效的数值，检查浏览器控制台是否有JavaScript错误

### Q: 量表评分不显示
**A**: 确保所有问题都已回答，检查是否正确初始化了问卷模块
